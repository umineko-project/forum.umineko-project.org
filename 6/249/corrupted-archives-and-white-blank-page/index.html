Hi,

First of all, make sure you use the latest 7-zip archiver as the previous versions are unfortunately known to awkward failures. You can get one from their site: https://www.7-zip.org.

Next, please note that the Chiru archives (just like all the other archives in fact) are multipart archives. All you need is to open the first file (chiru_01.07.17.7z.001), and all the subsequent files will be implicitly read by 7-zip itself during the extraction. That means, that all the parts (chiru_01.07.17.7z.002, chiru_01.07.17.7z.003, …) must be correctly named and put to the same directory as the 7z.001 part. If they are not, you will get the corrupted error just like you did above.

Last but not least, there always is a small chance for a download error, which is hard to notice when there are 6 files. Just in case, all the files but the last part (chiru_01.07.17.7z.001—chiru_01.07.17.7z.005) are 1072693248 bytes in length and chiru_01.07.17.7z.006 is 282676430 bytes. You could check file length in file properties.
