You are likely to face awful glitches, crashes, kernel panics or blue screens of death while honouring the flag of beta testing !
We expect you manage to properly report the issues in a NEW topic of this forum. Remember, one issue -> one topic, except the typos, which should be posted in typos topic.

Before you post a bug on this forum, make sure your computer and system conform to the system requirements. You are recommended to check the known issues thread, if your issue is known, there often is a workaround or a fix. If you have a crash or visual glitches, make sure your system is up-to-date(!), we may not accept bugs from outdated systems, and assure that you have working OpenGL 2.1+ drivers. To test OpenGL drivers you may use a simple win32 demo in the attachment (it should create a red-colour window).
init-demo.exe.zip
Topic should contain the following data to ease our understanding of your problem:
1. You must make sure to be using the latest engine and game script but even so, write engine filename and script date and filename here.
2. In case you got a crash, you should attach the crash logs. For example, if you are using Windows, attach engine_name.RPT file to your message. In case of OS X, post the log you may see after the program closes.
3. Describe the issue happened properly, step by step. Attach a screenshot if possible (and needed). Note, that you should either upload a screenshot to the forum or use any image host supporting https (e.g. https://img.bi, http://httpsimage.com whatever you find in google). If you prefer using screenshot-making tools, think of https://www.monosnap.com, it does offer https image hosting (if you correct the protocol of the url manually).
4. Write your bugreport in two languages if you can (English and Russian) to let any developer read it and help you.
5. It may well be helpful to post your hardware information here (esp. CPU, GPU, RAM, VRAM).

Always choose a sensible name of the topic describing the problem in brief. Titles like "I got a crash", "wtf is going on", "please, halp" are not informative. Add language tag(s) to the beginning of the topic title, e.g. [EN/RU] or [RU].

Очень вероятно, что вы встретитесь с жуткими глюками, крашами, паниками ядра или синими экранами во время нашего славного бета тестирования !
Мы очень надеемся, что вы сообщите обо всех найденных проблемах, создавая отдельную тему на каждую из них. Исключение составляют опечатки, которые следует размещать здесь.

Перед отправкой бага на форум убедитесь, что ваш компьютер и операционная система удовлетворяют системным требованиям. Рекомендуем просмотреть список известных проблем, так как там уже может находиться решение. Если происходит краш или графический глюк, стоит проверить обновлённость системы и драйверов(!), иначе, возможно, такую багу мы не примем. В системе должны находиться OpenGL 2.1+ совместимые драйвера. Для теста можно воспользоваться программой из вложения, она создаст окно с красным фоном.
init-demo.exe.zip
Созданная тема должна содержать следующую информацию о вашей проблеме:
1. Убедитесь, что вы используете самую последнюю версию скрипта и движка. Несмотря на очевидность этого, укажите имя файла движка, а также имя и дату изменения файла скрипта.
2. В случае краша следует разместить логи падения. Для Windows, к примеру, это файл совпадающий по имени с движком, но имеющий расширение RPT. Для OS X, текстовое сообщение, появляющееся после вылета программы.
3. Подробно опишите проблему, например, пошагово. Приложите скриншот, если это возможно, а также имеет смысл. Внимание, скриншот заливать либо на форум, либо на хост с поддержкой https (например, https://img.bi, http://httpsimage.com). На случай предпочтения программ для создания скриншотов, есть https://www.monosnap.com, однако надо будет вручную изменять http на https в полученной ссылке.
4. Если вы владеете английским (ну или японским), то описывайте проблему также и на них, так другие разработчики могут начать заниматься вашей проблемой раньше.
5. Информация о вашем ПК часто может помочь в решении проблемы, рекомендуем прикладывать её (хотя бы CPU, GPU, RAM, VRAM).

Выбирайте осмысленное название к теме, оно должно отражать суть проблемы. Всегда добавляйте тег языка к началу темы, например, [RU] или [EN/RU].
